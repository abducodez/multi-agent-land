# Codex Judge Rubric

Use this rubric before every major submission checkpoint.

## Product

- The first screen is the actual playable experience, not a landing page.
- The first 15 seconds produce a surprising AI-generated moment.
- The user can change the world, not only observe it.
- The app has a memorable identity and a clear demo beat.

## AI

- AI behavior is load-bearing and visible in the ledger.
- Each agent has a distinct role.
- Runtime model choices stay under 32B parameters.
- A <=4B mode is preserved for Tiny Titan eligibility.

## Architecture

- Event ledger remains the source of truth.
- Scenarios depend on core contracts, not on each other.
- Model providers are swappable by configuration or adapter.
- User input is treated as world data, not as privileged instructions.

## Documentation

- New architectural commitments have ADRs.
- Build progress has journal entries.
- The generated blog reflects the current project story.
- Demo and local development runbooks are current.

## Demo Readiness

- App starts from a clean checkout.
- Demo seed and injection are documented.
- Screenshots or video can be captured from the current UI.
- Known limitations are written down instead of hidden.
